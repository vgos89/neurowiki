## Phase 3.C — Big-rock initiatives (60–90+ days, V-decision-required)

### C1 — SPA pre-rendering or Vercel SSR (M2)

- **Impact:** high (most impactful long-term change for crawl reliability)
- **Effort:** L
- **V-gated:** yes — Class D architectural change, requires ADR
- **Phase 1 finding:** M2
- **Surfaces:** Vite config, Vercel project settings, `src/App.tsx` routing, `index.html`
- **What:** The current SPA architecture requires Googlebot to execute JavaScript to see per-route canonicals, titles, and structured data. Vercel's static hosting serves one index.html for all SPA routes. Two options: (a) Vite + vite-plugin-ssg pre-rendering: generate static HTML per route at build time — fits current stack, no hosting change, canonical and meta tags in HTML response for every route. (b) Vercel SSR (Edge Functions): serve per-route HTML server-side — more flexible for dynamic trial routes but requires runtime infrastructure. Either approach eliminates the JS-execution dependency for Googlebot and is the prerequisite for reliable canonical enforcement across all 120+ routes.
- **Acceptance check:** `curl https://neurowiki.ai/calculators/nihss | grep canonical` returns the correct per-route canonical without JS execution. `tsc --noEmit` passes. `npm run build` succeeds. All existing routes remain functional.
