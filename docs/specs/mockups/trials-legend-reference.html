Stage 3 — Intent
  • State A (Questions): white pill left, Catalog text slate-400 (#64748b). Default landing. CSS: .toggle-segment.active { background:white; color:#0f172a; box-shadow:0 1px 4px rgba(15,23,42,0.08); }.
  • State B (Hover over Catalog): NO pill, NO bg, NO slide. Only text color #64748b → #334155. CSS: .toggle-segment:hover:not(.active) { color:#334155; } transition 120ms ease. Faster than pill slide (220ms) so it responds immediately on cursor-over but doesn't flash on quick pass-through.
  • State C (Catalog active): white pill right. In implementation, the active class moves from left to right segment. Pill position transitions via background-color animation on the segments (simplest approach) or via a transform on a positioned pseudo-element. Both are correct; background transition avoids layout shift.
  • Section chevron (Stage 2): .section-chevron { transition: transform 200ms var(--ease-discovery); } .section-chevron.expanded { transform: rotate(180deg); }. Points up when section is open (showing arrow that would collapse it on click).
  • Search bar focus ring: border-color var(--cobalt) + box-shadow 0 0 0 4px var(--cobalt-soft). 150ms ease transition. Mirrors the cobalt-soft alpha overlay used in calculator-reference.html (lines 29–43 selected-option pattern).
  • Implementation priority order: toggle → trial cards → filter pills → search bar → hero H1 size responsiveness → section chevron → question items. Toggle and cards are load-bearing for the page; hero animation and section chevron are polish.
