## Semantic validity

**Five never-drift categories applied to the Option Y rendering rules:**

1. **Recommendation strength** — rules do not generate recommendation copy; they control NNT card rendering and color/label state. Pass.
2. **Action verbs** — none generated by these rules. Pass.
3. **Qualifiers and gates** — NNT suppression targets the NNT card only; existing population-gate prose in `applicability`, `bedsidePearl`, `howToInterpret` is structurally separate and must remain unaffected (see Required follow-up #4). Pass with condition #4.
4. **Certainty markers** — four failures:
   - `noninferiority-established` rendered as bare "positive" risks superiority inference (certainty upgrade drift). **Condition #2.**
   - `noninferiority-not-established` rendered as bare "negative" misframes NI failure as null result (certainty marker loss). **Condition #2.**
   - `harm-stopped` rendered as generic "negative" launders an active harm signal into a null result — category-4 drift. **Mandatory condition #1.**
   - Bayesian annotation "NNT derived from posterior probability difference" misattributes NNT derivation (incorrect certainty-source claim). **Condition #3.**
5. **Temporal constraints** — none affected by these rules. Pass.

**Answers to the five clinical questions:**

1. **Suppress NNT for `ordinal-shift` + `met` (SELECT-2, ANGEL-ASPECT)?** Clinically appropriate to suppress the NNT *card* — no pre-specified binary dichotomization exists on the ordinal scale. The prose in `bedsidePearl`/`howToInterpret` for these trials carries dichotomized secondary-endpoint NNT equivalents and must not be collateral-suppressed (see condition #4). Approved.

2. **`noninferiority-established` → "positive" + no NNT — right combination?** Half-right. No NNT is correct. "Positive" without an NI qualifier risks superiority inference. Labels for `noninferiority-established` must read "non-inferiority established" (not "positive" alone). This was the explicit Wave 2 follow-up (clinical-wave2-evt-schema.md line 56) and must close in this wave. **Mandatory condition #2.**

3. **`harm-stopped` → "negative" — adequate?** No. This is a never-drift category-4 violation. `harm-stopped` trials (NOR-TEST 2 Part A in the current EVT/IVT dataset, SAMMPRIS in Batch 3) must render with a distinct harm state — warning/danger visual treatment and label "stopped for harm" or equivalent. Collapsing to "primary endpoint not met" launders the safety signal. **Mandatory condition #1.**

4. **DAWN Bayesian NNT — annotation wording accurate?** The NNT value (~2.8 from ARD 35.5pp) is correct. The annotation wording is not: NNT is derived from the *observed proportions* (48.6% vs 13.1% mRS 0-2), not from the posterior probability. The posterior probability (>99.9%) is the decision rule for stopping, not the effect-size source. Required wording: "(Bayesian adaptive trial — superiority established by posterior probability >0.999; NNT calculated from observed proportions.)" **Mandatory condition #3.**

5. **`noninferiority-not-established` → generic "negative"?** Not sufficient without a qualifier. Labels must read "did not establish non-inferiority" or "non-inferiority not met." Note also: NOR-TEST is tagged `primaryDesign: 'noninferiority'` + `primaryResult: 'noninferiority-not-established'`, but its `doesNotProve` field says "It was a superiority trial, not an NI trial." This data inconsistency is a Wave 2 issue to fix separately but will surface via Wave 3 labels; flagged as a non-blocking data follow-up. **Mandatory condition #2.**

## Rationale

The Option Y rules are clinically defensible in direction: NNT suppression for ordinal-shift, NI, and dose-finding-safety designs is correct; NNT display for Bayesian superiority (DAWN) is correct; schema-driven classification replaces imprecise string-sniffing. However, four label and annotation choices fail the never-drift standard and must be fixed in the implementation before merge: harm-stopped needs its own visual/label state (not generic negative), NI-related labels need explicit NI qualifiers, and the Bayesian annotation must correctly attribute NNT derivation. All four are renderer-string fixes requiring no new claim authoring; they must be resolved before this PR merges.

## Required follow-ups

1. **Mandatory before merge — `harm-stopped` distinct rendering.** Rule 2 must split into two states: (a) generic-negative for `not-met`, `futility-stopped`, `terminated-administrative`; (b) harm-specific for `harm-stopped` — distinct visual treatment (warning/danger) and label "stopped for harm" or "primary endpoint not met — stopped for excess harm." NOR-TEST 2 Part A is the current canary; SAMMPRIS (Batch 3) will be the second.

2. **Mandatory before merge — NI qualifier labels.** `noninferiority-established` must render with label "non-inferiority established" (not generic positive). `noninferiority-not-established` must render with "did not establish non-inferiority" (not generic negative). This closes the explicit follow-up from clinical-wave2-evt-schema.md line 56.

3. **Mandatory before merge — Bayesian annotation wording.** Replace "NNT derived from posterior probability difference" with: "(Bayesian adaptive trial — superiority established by posterior probability >0.999; NNT calculated from observed proportions.)"

4. **Mandatory before merge — verify NNT suppression targets card only.** Confirm on SELECT-2, ANGEL-ASPECT, ESCAPE-MeVO, DISTAL, and the NI trials that prose surfaces (`bedsidePearl`, `howToInterpret`, `bottomLineSummary`) that reference NNT-equivalent figures are not affected by Rule 1 NNT-card suppression.

5. **Non-blocking — data inconsistency: NOR-TEST tagging.** NOR-TEST is tagged `primaryDesign: 'noninferiority'` + `primaryResult: 'noninferiority-not-established'` but `doesNotProve` says "It was a superiority trial, not an NI trial." Open a task to reconcile — likely reclassify as `binary-superiority` + `not-met`. This is a Wave 2 data correction, not a Wave 3 renderer issue.

6. **Non-blocking — composition-site claim tagging.** When Rule 3's Bayesian annotation copy is finalized, it is a §13.3 Phase 3 composed clinical surface and requires a `claimId` at the composition site (e.g., `dawn-bayesian-nnt-annotation`). Add when wording is locked.
