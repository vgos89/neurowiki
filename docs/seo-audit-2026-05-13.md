# SEO Site Audit — NeuroWiki

**Date:** 2026-05-13
**Reviewer:** seo-specialist (model: claude-sonnet-4-6)
**Production URL:** https://neurowiki.ai
**Status:** findings only — no code changes

---

## Executive summary

NeuroWiki has a solid SEO foundation: every route has routeManifest.ts metadata, the Seo.tsx component injects per-route title/canonical/OG/JSON-LD on navigation, robots.txt is clean, and structured data schemas cover all major surface types (MedicalWebPage, SoftwareApplication, MedicalScholarlyArticle, BreadcrumbList, FAQPage). The two critical blockers are: (1) the static sitemap.xml uses wrong URL paths for six pathway routes — Googlebot is currently submitting URLs that 404, costing crawl budget and suppressing those pages from indexing; and (2) `TrialPageNew.tsx` uses `<h1>` for individual stat-card labels, producing 30–60 H1 tags per trial page, which signals poor document structure to crawlers. Both are fixable without clinical content changes.

## Findings — High priority

### H1 — sitemap.xml contains wrong paths for pathway routes

- **Surface:** `public/sitemap.xml` lines 36–75
- **Current state:** Six pathway entries use `/calculators/` prefix:
  - `https://neurowiki.ai/calculators/evt-pathway` (actual route: `/pathways/evt`)
  - `https://neurowiki.ai/calculators/elan-pathway` (actual route: `/pathways/elan-pathway`)
  - `https://neurowiki.ai/calculators/late-window-ivt` (actual route: `/pathways/late-window-ivt`)
  - `https://neurowiki.ai/calculators/se-pathway` (actual route: `/pathways/se-pathway`)
  - `https://neurowiki.ai/calculators/migraine-pathway` (actual route: `/pathways/migraine-pathway`)
  - `https://neurowiki.ai/calculators/gca-pathway` (actual route: `/pathways/gca-pathway`)
- **Expected state:** All URLs match actual SPA routes from routeManifest.ts under `/pathways/`.
- **Why it matters:** Google submits these URLs from the sitemap and receives the React app's 404 (or home redirect). Pages are not indexed. Crawl budget is wasted. Pathway pages that have FAQ structured data and per-route schema are invisible to search.
- **Fix complexity:** S — edit six `<loc>` values in sitemap.xml.

### H2 — Multiple H1 elements per trial page (TrialPageNew.tsx)

- **Surface:** `src/pages/trials/TrialPageNew.tsx` lines 404, 476, 937, 1020, 1096, 1172, 1256, 1340, 1417, 1493, 1576, 1660, 1741, 1828, 1904 … (65+ total H1 elements)
- **Current state:** Individual stat cards (primary outcome, secondary outcome, safety results, NNT panels) use `<h1>` for their display labels. A single rendered trial page has approximately 30–60 `<h1>` elements depending on trial size.
- **Expected state:** One `<h1>` per page containing the trial name. Stat labels should be `<h3>` or `<p>` with `font-weight` styling. Existing H1 at line 404 (`{catalogTrial.name}`) is the correct and only page-level H1; the archetype-A renders at line 476.
- **Why it matters:** Google's documentation states it uses H1 to understand page topic. Multiple H1s create ambiguity about the primary topic, diluting the topical signal. Screen readers also rely on H1 for landmark navigation.
- **Fix complexity:** M — replace `<h1>` with `<h3>` (or `<p>`) inside each stat-card render function in TrialPageNew.tsx.

### H3 — Duplicate page titles across two routes

- **Surface:** `src/config/routeManifest.ts` lines 384 and 447
- **Current state:** Both `/pathways/stroke-code` and `/guide/stroke-basics` share the identical title: `"Stroke Code Protocol — Acute Stroke Workflow for Residents | NeuroWiki"`.
- **Expected state:** Each route has a unique `<title>`. The stroke-basics route should reflect its content (stroke code reference guide) rather than mirroring the interactive pathway title.
- **Why it matters:** Google treats identical titles on different URLs as a signal of thin or duplicate content. One page may be suppressed from SERPs in favor of the other. Duplicate titles also make SERP click-through ambiguous.
- **Fix complexity:** S — update one title string in routeManifest.ts.

## Findings — Medium priority

### M1 — Sitemap missing routes that have includeInSitemap: true in routeManifest.ts

- **Surface:** `public/sitemap.xml` vs `src/config/routeManifest.ts`
- **Current state:** The following routes have `includeInSitemap: true` in routeManifest.ts but are absent from sitemap.xml:
  - `/calculators/chads-vasc` (CHA₂DS₂-VASc calculator)
  - `/pathways/stroke-code` (Stroke Code pathway)
  - `/guide/aha-2026-guideline` (AHA 2026 Guideline Mindmap)
  Conversely, `/pathways/migraine-pathway` and `/pathways/gca-pathway` have `includeInSitemap` unset (no property) but were historically in the sitemap under wrong paths.
- **Expected state:** sitemap.xml matches `STATIC_SITEMAP_ROUTES` from routeManifest.ts exactly. `sitemapRoutes.ts` exports `SITEMAP_ROUTES` but this is not used to auto-generate `public/sitemap.xml` — the file is manually maintained and has drifted.
- **Why it matters:** Missing sitemap entries slow discovery of new content. Search engines rely on the sitemap as the authoritative list of indexable URLs.
- **Fix complexity:** M — either regenerate sitemap.xml from SITEMAP_ROUTES at build time, or manually sync it.

### M2 — SPA-only canonical tags: JS must execute for per-route canonicals

- **Surface:** `index.html` line 30 + `src/components/Seo.tsx` line 41
- **Current state:** index.html contains a static canonical `<link rel="canonical" href="https://neurowiki.ai">` for the home page. Per-route canonicals are injected by Seo.tsx via `document.head` manipulation on navigation. Googlebot must execute JavaScript to see any canonical other than the home URL.
- **Expected state:** Either server-side rendering that serves per-route canonical tags in the HTML response, or pre-rendered HTML per route at build time. Alternatively, the static canonical in index.html should be removed so Googlebot falls back to the URL rather than seeing `neurowiki.ai` as the canonical for every page before JS runs.
- **Why it matters:** If Googlebot does not fully execute the JS (e.g., slow rendering, quota), the home canonical could be applied to all pages, causing all URLs to be treated as duplicates of the homepage. Vercel's static hosting serves one index.html for all SPA routes.
- **Fix complexity:** L — SSR or pre-rendering is a Class D infrastructure change. Removing the static home canonical from index.html is S, but the underlying SPA canonical gap persists.

### M3 — /pathways hub has no JSON-LD schema

- **Surface:** `src/seo/schema.ts` line 621 — `if (pathname === '/pathways') return null;`
- **Current state:** The pathways hub at `/pathways` explicitly returns null schema. All other hub pages (home, /calculators, /guide, /trials) have defined schemas.
- **Expected state:** `/pathways` should have a WebPage or CollectionPage schema with BreadcrumbList and ItemList of child pathways, matching the pattern for other hubs.
- **Why it matters:** The pathways hub is a high-traffic discovery page. Missing structured data means no rich result eligibility and a weaker topical signal.
- **Fix complexity:** S — add a PATHWAYS_HUB_SCHEMA constant in schema.ts and wire it to the `/pathways` branch.

### M4 — Several title tags exceed 60-character guideline

- **Surface:** `src/config/routeManifest.ts`
- **Current state:** The following titles exceed 60 characters (Googlebot truncates at ~60 chars in SERPs):
  - ASPECTS Score: `"ASPECTS Score Calculator — Alberta Stroke Program Early CT Score | NeuroWiki"` (78 chars)
  - E/M Billing: `"E/M Billing Calculator — CPT Code 99202–99215 | MDM & Time-Based | NeuroWiki"` (79 chars)
  - Boston Criteria: `"Boston Criteria 2.0 for CAA — Cerebral Amyloid Angiopathy | NeuroWiki"` (71 chars)
  - Heidelberg: `"Heidelberg Bleeding Classification — Hemorrhagic Transformation | NeuroWiki"` (77 chars)
  - CHA₂DS₂-VASc: `"CHA₂DS₂-VASc Score — AF Stroke Risk Calculator | NeuroWiki"` (60 chars — borderline)
  - Late Window IVT: `"Late Window IVT — Wake-Up Stroke & Thrombolysis Eligibility | NeuroWiki"` (73 chars)
  - Status Epilepticus Management: `"Status Epilepticus Management Guide — First Line to Refractory | NeuroWiki"` (76 chars)
  - ICH Management: `"ICH Management — Intracerebral Hemorrhage Protocol | NeuroWiki"` (63 chars)
  - EVT Eligibility: `"EVT Eligibility Tool — Thrombectomy Decision Support | NeuroWiki"` (65 chars)
  - Stroke Code: `"Stroke Code Protocol — Acute Stroke Workflow for Residents | NeuroWiki"` (72 chars)
- **Expected state:** All titles ≤60 characters, or at minimum ≤70 characters (Google's practical cutoff before truncation in desktop SERPs).
- **Why it matters:** Truncated titles in SERPs reduce click-through rate. Truncation at "…" often cuts off the brand name "NeuroWiki."
- **Fix complexity:** S per title — copywriting decision per route, no code change required beyond string edits.

### M5 — og:image and twitter:image absent for most non-home routes

- **Surface:** `src/config/routeManifest.ts` — `image` field; `src/components/Seo.tsx` lines 50, 56
- **Current state:** Only routes explicitly setting `image: DEFAULT_IMAGE` in their meta block have OG/Twitter images (home, calculators hub, guide hub, trials hub, a few guide articles). Calculator routes, pathway routes, and all trial routes generated by `buildTrialMeta` in routeMeta.ts do not set `image`, so `setMeta('og:image', ...)` is skipped.
- **Expected state:** All public routes should have `og:image` pointing to at least the default OG image. Seo.tsx should fall back to DEFAULT_IMAGE if `meta.image` is undefined.
- **Why it matters:** Social shares of calculator and trial pages will show no preview image on LinkedIn, Twitter/X, and iMessage. This reduces click-through from social referral.
- **Fix complexity:** S — add fallback `const image = meta.image ?? DEFAULT_META.image` in Seo.tsx before the setMeta calls.

### M6 — Extensive link graph orphans: trial nodes have no inbound references

- **Surface:** `docs/link-graph.json` — trial nodes
- **Current state:** 23 of the 29 trial nodes in link-graph.json have `"referencedBy": []`:
  `trial/extend-trial`, `trial/wake-up-trial`, `trial/escape-mevo-trial`, `trial/elan-study`, `trial/chance-trial`, `trial/point-trial`, `trial/socrates-trial`, `trial/sps3-trial`, `trial/sparcl-trial`, `trial/thales-trial`, `trial/eagle-trial`, `trial/best-msu-trial`, `trial/act-trial`, `trial/aramis-trial`, `trial/nor-test-trial`, `trial/nor-test-2-part-a-trial`, `trial/prisms-trial`, `trial/prost-trial`, `trial/prost-2-trial`, `trial/raise-trial`, `trial/taste-trial`, `trial/thaws-trial`, `trial/trace-2-trial`, `trial/weave-trial`.
  Additionally `hub/guide` has `"referencedBy": []` — no page links to the guide hub.
  `calc/abcd2` has both empty `references` and `referencedBy`.
- **Expected state:** Every node has at least one inbound reference. Trial pages referenced in pathways or guide articles should have those articles listed in `referencedBy`. Guide hub should be referenced from at least the home/trials hub. ABCD2 calculator should be referenced from the TIA-related pathway or guide.
- **Why it matters:** Orphan pages receive no PageRank flow from internal links, limiting their ranking potential. The link graph is also the source of truth for LINK_GRAPH.md.

### M7 — Large number of article/guide nodes have no outbound references (dead-end nodes)

- **Surface:** `docs/link-graph.json` — article nodes
- **Current state:** 13 of 17 guide article nodes have `"references": []`:
  `article/thrombectomy`, `article/acute-stroke-mgmt`, `article/aha-2026-guideline`, `article/status-epilepticus`, `article/seizure-workup`, `article/altered-mental-status`, `article/meningitis`, `article/headache-workup`, `article/vertigo`, `article/weakness-workup`, `article/gbs`, `article/myasthenia-gravis`, `article/multiple-sclerosis`.
  `article/stroke-basics` also has no outbound references.
- **Expected state:** Guide articles should reference related calculators, trials, and pathways. E.g., `article/thrombectomy` should reference `calc/nihss`, the EVT pathway, and DAWN/DEFUSE-3 trials.
- **Why it matters:** Internal linking is a core ranking factor. Dead-end articles do not pass link equity and miss the SEO benefit of cross-linking contextually related tools.
